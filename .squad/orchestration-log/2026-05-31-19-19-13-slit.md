## Summary

Fixed drag-and-drop failure in WPF UI. Drag handlers (DragEnter, DragOver, DragLeave, Drop) were only wired to the Window element, but WPF's OLE drag-drop system targets the innermost AllowDrop="True" element (the DropZoneBorder). Without handlers on the border, WPF never set e.Effects = DragDropEffects.Copy, causing the cursor and silent Drop rejection.

## Root Cause

Commit ba4c336 added AllowDrop="True" to DropZoneBorder but did not attach drag handlers. This made the border the OLE drop target with no confirmation logic — the Window-level handlers were never reached because OLE checked the border first and found no support for copy effect.

## Fix

Added four attributes to DropZoneBorder in MainWindow.xaml:
- DragEnter="Window_DragEnter"
- DragOver="Window_DragOver"
- DragLeave="Window_DragLeave"
- Drop="Window_Drop"

Reused existing handler methods from code-behind. No new code, no Core changes.
